Administrative details

Our client is Kjell-Inge Johansen. He is a friend of Ingvild in the group, and they met playing the Ingress mobile game.

Kjell-Inge is a truck driver, with a growing side business shooting drone footage. He would like a webpage to display his cinematic drone videos and images, but also to get more drone shoot commissions.

Changes made to contact.html

The changes made to contact.html are in that some of the repetitive elements like facebook and instagram links were not present twice because we felt that it was too repetitive. We also decided that it wasn't necessary to list the owner's telephone number because it was his private number. This is something he can change at a later date.

Changes to drones_and_cameras.html

We wanted to create a page which was welcoming to new potential customers and displayed professionalism and knowledge of the field. We implemented stunning pictures of the drones Kjell-Inge currently use.

Changes to index.html

The index is quite similar to the draft. We wanted to display a video in the background, which we succeeded with. We added a third “infobox” with information aimed at people selling cars, houses and alike.

Changes to portfolio.html:

Portfolio.html didn’t change much from the design. Some differences to font sizes, minor placement changes of text and images. The major difference from the design description is the support for videos in the same way as images. It displays a full width video, but the image series created by portfolio.js and lightbox.js fullscreen view does not support videos. In addition the video player in the design had custom controls, and these were not implemented.

Changes to Portfolio.js:

Originally portfolio.js was the javascript for all the portfolio / gallery / carousel functionality. It was split into two files, portfolio.js and lightbox.js. Portfolio.js houses lists of images, setup variables for lightbox.js and functionality to render images to portfolio.html and bind these images so that clicking on them opens a lightbox.

Changes to lightbox.js:

Originally lightbox.js was a part of portfolio.js. With the goal of enhancing the way the website can showcase Kjell-Inges photos and videos, there was a bit of uncertainty of what this functionality would look like. It was uncertain how hard this functionality would be to implement as well. Lightbox.js ended up being a fullscreen image viewer. It opens by calling a javascript function and it takes up the whole viewport, hiding everything else. It takes in a set of images in a javascript list of objects. Each object contains an image filename and an image description. It displays one large image on top, and all the other images are displayed as thumbnails at the bottom in a horizontal list. The active thumbnail is highlighted and brought to the left of the list. Users can navigate left and right, either using buttons on the website, or the keyboard arrow keys, or click any thumbnail to set it as the active one. Users can exit the lightbox by pressing escape or clicking the exit button in the top right. It also has two features to show how many images there are in total, as well as showing where in the set you currently are: a bar at the bottom and text at the top. It was supposed to support videos, but time constraints and other features were prioritized.

Changes made to validate.js:

The form were in the beginning supposed to be used to validate just some of the information written in the contact.html form. We also decided that it was better to not lock out user who did not write a proper telephone number. Thus we decided that is was better to suggest what the user were supposed to write instead of locking the user out of sending the form.

Changes made to services_and_prices.html

The prices and services-page has changed quite a bit. The old version felt a bit boring and bland, so more of the design elements from index and others were used, including the separators and highlighters. We also added a price calculator so that users can get a price estimate. We were initially a bit unsure whether this was useful for Kjell-Inge, but this way, he can decide if he wants to keep it or not.

Changes made to terms_of_services.html

The Terms of Services is not changed in particular, and it is strictly an infopage made for people who want more information on how the site functions.
